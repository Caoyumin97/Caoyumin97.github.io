## 从 OpenClaw 说起

最近 OpenClaw 引发的讨论铺天盖地。作为小米供应链管理组的 AI 应用负责人，也是一个写了几年算法的工程师，我想从自己的视角聊聊 AI Agent 的演化方向，以及它可能给行业和社会带来的真实变化——而不只是又一轮 hype cycle 里的噪音。

OpenClaw 本质上在做的事情，是把大语言模型从"对话接口"推向"执行引擎"。这不是一个渐进式的改进，而是范式的跃迁。对话意味着人类仍然是执行主体，模型只是顾问；而 Agent 意味着模型本身成为执行主体，人类退到监督者和决策者的位置。

## Agent 的真实能力边界

在供应链场景里，我每天都在和 Agent 的能力边界打交道。说实话，现阶段的 Agent 并没有很多人想象的那么"智能"，但也比很多人以为的要有用得多。

**Agent 已经很擅长的事情：**

- 跨系统的信息聚合与摘要。以前需要一个人在 ERP、WMS、TMS 之间来回切换拼凑的信息，Agent 可以在几秒内完成。
- 规则明确的决策执行。当补货策略、异常预警的逻辑可以被清晰地描述时，Agent 执行得又快又稳。
- 自然语言到结构化查询的转换。业务同事不用再学 SQL，直接用中文问"上个月华东仓的周转天数是多少"。

**Agent 仍然挣扎的事情：**

- 需要深度领域知识的模糊决策。比如某个供应商交期延迟了，到底是催货、换供应商、还是调整生产计划，这取决于大量隐性知识和上下文。
- 多步骤的复杂协调。涉及到跨部门、跨时区的协作流程，Agent 还是会在中间环节出错或丢失上下文。
- 对"不确定性"的合理处理。供应链的核心挑战就是不确定性，而现在的 Agent 要么过度自信，要么在不确定时直接停摆。

## 行业会怎么变

从供应链管理的角度看，我认为 AI Agent 会带来三个层面的变化：

**第一层：工具层面的变化。** 这已经在发生了。各种 Copilot 类的产品在提升个体效率。但说实话，这一层的变化没有想象中那么剧烈——工具再好，业务流程和组织架构不变，整体效率的提升是有上限的。

**第二层：流程层面的重构。** 这是接下来两到三年最重要的事情。当 Agent 可以可靠地执行一整条业务链路（比如从需求预测到自动补货到异常处理），很多传统的审批流程、人工复核环节就会被重新设计。不是简单地"用 AI 替代人"，而是整个流程会被重新想象。

**第三层：组织形态的演化。** 这更长远，但方向已经能看到了。如果一个三人团队加上一组 Agent 能完成以前十个人做的事，那组织结构、汇报关系、绩效评估都得重新来过。小米内部我们已经在尝试一些"人+Agent"的混合编组，效果和挑战都很真实。

## 社会影响：不只是失业问题

很多讨论把 AI 对社会的影响简化成"谁会失业"，这太狭隘了。更值得思考的是：

**知识的去中心化。** 以前，一个资深供应链专家脑子里的经验是极其宝贵的——因为它无法被轻易传递。现在，Agent 可以把这些知识编码化、可复用化。这对个体专家的议价能力是挑战，但对整个系统的效率是巨大的提升。

**决策权的重新分配。** 当一线操作人员可以通过 Agent 获得以前只有管理层才有的信息和分析能力时，决策权自然会下沉。这是好事，但也意味着组织需要新的协调机制。

**信任的建立。** 这是最被低估的问题。我们团队花在"让业务方信任 Agent 的输出"上的时间，比花在技术开发上的时间还多。技术问题终会被解决，信任问题才是真正的慢变量。

## 一些务实的想法

我不太喜欢那种"AI 将改变一切"或者"AI 就是泡沫"的极端叙事。现实永远在中间。

作为一个每天都在一线落地 AI 的人，我的观察是：**Agent 的真正价值不在于它多"智能"，而在于它能让"信息-决策-执行"的链路变短。** 在供应链这种信息密度极高、决策链路极长的领域，这本身就是巨大的价值。

但同时，我们也要警惕把 Agent 当成万能钥匙。每一个成功的 Agent 应用背后，都有大量的领域建模、流程设计、异常处理的工作。这些工作不性感，但离开它们，Agent 就是一个很贵的聊天机器人。

Hype 会过去，真正的变化会留下来。作为从业者，我们能做的就是在噪音中找到信号，然后一步一步把它做实。
